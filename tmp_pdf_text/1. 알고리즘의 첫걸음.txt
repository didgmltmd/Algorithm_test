알콰리즈미(Al-Khwarizmi)
알고리즘의 유래
• 알고리즘(Algorithm)이라는 용어는 9세기 페르시아의
수학자 무함마드 이븐 무사 알콰리즈미의 이름에서 유래
• 그의 저서가 라틴어로 번역되면서 'Dixit Algorizmi'(알
콰리즈미가 말하기를)라는 문구로 시작, 시간이 흐르며 수
치 계산법을 뜻하는 'Algorismus'로 변했음
• 최초의 알고리즘 : BC 300년 경 유클리드의 최대공약수(GCD) 알고리즘

학문적 기여
대수학의 아버지
• 알콰리즈미는 0을 포함한 인도-아라비아 숫자를 유럽에
전파하여 현대 수학의 기틀을 마련
• 그의 저서 “복원과 대비의 계산법”(Al-Jabr)은 현대
수학의 한 분야인 '대수학(Algebra)'의 어원
• 이처럼 알고리즘은 단순히 계산을 넘어 문제를 해결하는
'체계적 방법'으로 발전하기 시작

알고리즘의 5가지 조건
입력 (Input)
외부에서 제공되는 데이터가 0개 이상 존재해야 함
출력 (Output)
적어도 1개 이상의 결과가 출력되어야 함
명확성 (Definiteness)
각 단계는 모호하지 않고 실행 가능하며 명확한 의미를 가져
야 함
유한성 (Finiteness)
알고리즘은 일정한 수의 단계를 거친 후에는 반드시 종료되
어야 함
효율성 (Effectiveness)
모든 명령은 종이와 연필만으로도 실행할 수 있을 정도로 단
순하고 구체적이어야 함
100%
실행 가능성
5가지 조건을 충족할 때 '올바른 알고리즘'

def search(arr, x):
for i in range(len(arr)):
if arr[i] == x:
return i
return -1

순차 탐색의 최적화 및 활용
자기 구성 리스트 (Self-organizing): 자주 찾는 데이터를 리스트의 앞쪽으로
이동시켜 평균 탐색 시간을 단축할 수 있음
소규모 데이터 최적: 데이터 양이 적을 때는 이진 탐색을 위한 정렬 비용보다 순차
탐색 비용이 더 저렴할 수 있음
연결 리스트 활용: 인덱스 접근이 불가능한 연결 리스트(Linked List) 구조에서
는 순차 탐색이 가장 기본
안정성: 단순한 구조 덕분에 버그 발생 확률이 낮고 신뢰성이 높음


--- PAGE 27 ---
순차 탐색이 가장 유리한 순간
데이터 양이 매우 적을 때: 이진 탐색처럼 정렬하는 오버헤드보다 그냥 찾는 것이 더
빠름
데이터가 정렬되어 있지 않을 때: 정렬을 위한 추가 연산이 필요 없어 즉시 탐색이
가능
연결 리스트(Linked List) 구조일 때: 인덱스로 바로 접근할 수 없는 구조에서 순차
접근은 필수적
찾는 데이터가 앞쪽에 있을 확률이 높을 때: 평균 탐색 시간이 획기적으로 줄어듬
