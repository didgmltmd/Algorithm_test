2.1 알고리즘이란
 알고리즘
• 문제를 해결하는 단계적 절차 또는 방법
• 여기서 주어지는 문제는 컴퓨터를 이용하여 해결할 수 있어야 함
• 알고리즘에는 입력이 주어지고, 알고리즘은 수행한 결과인 해 (또는 답)를 출력함


--- PAGE 5 ---
알고리즘의 일반적 특성
 정확성
• 알고리즘은 주어진 입력에 대해 올바른 해를 주어야(랜덤 알고리즘은 예외)
 수행성
• 알고리즘의 각 단계는 컴퓨터에서 수행 가능해야 함
 유한성
• 알고리즘은 유한 시간 내에 종료되어야 함
 효율성
• 알고리즘은 효율적일수록 그 가치가 높아짐

• 기원전 300년경에 만들어진 가장 오래된 알고리즘
• 최대공약수란 2개의 자연수의 공약수들 중에서 가장 큰 수
2개의 자연수의 최대공약수는 큰 수에서 작은 수를 뺀 수와 작은 수와의
최대공약수와 같다.

최대공약수(24, 14)
최대공약수(24, 14)
= 최대공약수(24-14, 14) =  최대공약수(10, 14)
= 최대공약수(14-10, 10) =  최대공약수(4, 10)
= 최대공약수(10-4, 4) =  최대공약수(6, 4)
= 최대공약수(6-4, 4) =  최대공약수(2, 4)
= 최대공약수(4-2, 2) =  최대공약수(2, 2)
= 최대공약수(2-2, 2) =  최대공약수(0, 2)
= 2

Euclid(a, b)
입력: 정수 a, b;  단, a≥b≥0
출력: 최대공약수(a, b)
1. if b==0 return a
2. return Euclid(b, a mod b)
최대공약수(24, 14)
• Line 1: b==14이므로 if-조건이 ‘거짓‘
• Line 2: Euclid(14, 24 mod 14) = Euclid(14, 10) 호출
• Line 1: b==10이므로 if-조건이 ‘거짓‘
• Line 2: Euclid(10, 14 mod 10) = Euclid(10, 4) 호출
• Line 1: b==4이므로 if-조건이 ‘거짓‘
• Line 2: Euclid(4, 10 mod 4) = Euclid(4, 2) 호출
• Line 1: b==2이므로 if-조건이 ‘거짓‘
• Line 2: Euclid(2, 4 mod 2) = Euclid(2, 0) 호출
• Line 1: b==0이므로 if-조건이 ‘참’이 되어a=2를 최종 리턴
Mod(모듈러) : 양변의 두 수를 서로 나누어서 몫을
제외한 나머지 값을 구하는 나머지
연산 의미
- 문법 : b = mod(a,m)
- 수식 : a mod b

유클리드의 최대공약수 알고리즘
def get_gcd_recursive(a, b): 
# 나누어 떨어져서 나머지(b)가 0이 되면 a를 반환합니다. 
if b == 0:
return a 
# 그렇지 않으면 b와 'a를 b로 나눈 나머지(mod)'를 가지고 다시 함수를 실행합니다. 
else: 
return get_gcd_recursive(b, a % b) 
# 테스트
num1 = 60 
num2 = 48 
print(f"{num1}과 {num2}의 최대공약수: {get_gcd_recursive(num1, num2)}")

유클리드의 최대공약수 알고리즘
def get_gcd_with_mod(a, b): 
# b가 0이 아닐 때까지 반복합니다. 
while b != 0: 
# a를 b로 나눈 나머지(mod)를 구합니다. 
remainder = a % b 
# a에는 b값을, b에는 나머지 값을 덮어씌웁니다. 
a = b 
b = remainder 
# 파이썬에서는 위 세 줄을 "a, b = b, a % b" 한 줄로 줄여 쓸 수도 있습니다.
return a   # b가 0이 되면, 그때의 a가 최대공약수입니다.
# 테스트
num1 = 60 
num2 = 48 
print(f"{num1}과 {num2}의 최대공약수: {get_gcd_with_mod(num1, num2)}")

최대 숫자 찾기 문제를 위한 알고리즘
 최대 숫자 찾기
• 카드의 숫자를 하나씩 비교하면서 본 숫자들 중에서 가장 큰 숫자를 기억해가며
찾는다.
• 마지막 카드의 숫자를 본 후에, 머릿속에 기억된 가장 큰 숫자가 적힌 카드를
바닥에서 집어 든다.


--- PAGE 15 ---
보통 말로 표현된 알고리즘
1. 첫 카드의 숫자를 읽고 머릿속에 기억해 둔다.
2. 다음 카드의 숫자를 읽고, 그 숫자를 머릿속의 숫자와 비교한다.
3. 비교 후 큰 숫자를 머릿속에 기억해 둔다.
4. 다음에 읽을 카드가 남아있으면 line 2로 간다.
5. 머릿속에 기억된 숫자가 최대 숫자이다.

--- PAGE 16 ---
의사 코드로 표현된 알고리즘
배열A 에 10개의 숫자가 있다면
1. max = A[0]
2. for i = 1 to 9
3. if (A[i] > max) max = A[i]
4. return max

최대 숫자 찾기 문제 알고리즘
def find_max_with_loop(numbers): 
# 빈 리스트가 들어올 경우를 대비한 예외 처리
if not numbers: 
return "배열이 비어있습니다." 
# 1. 배열의 첫 번째 숫자를 임시 최댓값으로 설정합니다. 
max_num = numbers[0] 
# 2. 배열의 두 번째 숫자부터 끝까지 순회하며 하나씩 확인합니다. 
# numbers[1:] 는 첫 번째 요소를 제외한 나머지 리스트를 의미합니다.
for current_num in numbers[1:]: 
# 3. 현재 숫자가 임시 최댓값보다 크다면? 
if current_num > max_num: 
# 최댓값을 현재 숫자로 갱신합니다. 
max_num = current_num
# 반복문이 모두 끝나면 최종적으로 가장 큰 수가 max_num에 남아있게 됩니다.
return max_num

2. 정복 & 결합
하위 문제를 재귀적으로 해결하고(Conquer), 하위
문제의 규모가 나눌 수 없는 단위가 되면 탈출 조건을

 동일한 문제라도 입력 데이터의 특성에 따라 알고리즘의 성능이 달라질 수 있음

• 순차 탐색으로 찾는 경우에 숫자 비교가 기본적인 연산이고, 총 비교 횟수는 9번

 최악 경우 분석(Worst-case Analysis)

 최선 경우 분석(Best-case Analysis)

길고, 다른 하나는 짧으며, 수행 시간이 짧은 연산은 많이 수행되고 수행 시간이 긴

O(Big-Oh)-표기
점근적 상한 (Upper Bound)
• 알고리즘의 실행 시간이 "아무리 나빠도 이보다는 좋다"
는 것을 의미
• 일반적으로 알고리즘의 최악의 경우(Worst Case)를
표현하는 데 사용됨


--- PAGE 83 ---
O(Big-Oh)-표기
 O-표기의 정의
• 모든 n ≥ n0에 대해서 f(n) ≤ cg(n)이 성립하는 양의 상수 c와 n0가 존재하면, f(n) = O(g(n))
 O-표기의 의미
• n0와 같거나 큰 모든 n (즉, n0 이후의 모든 n)에 대해서 f(n)이 cg(n)보다 크지 않다는 것
 f(n) = O(g(n))은 n0 보다 큰 모든 n 대해서 f(n)이 양의 상수를 곱한 g(n)에
미치지 못한다는 뜻
 g(n)을 f(n)의 상한(Upper Bound)이라 함

--- PAGE 84 ---
O(Big-Oh)-표기
 f(n) = O(g(n))
• n이 증가함에 따라 O(g(n))이 점근적 상한이라는 것 (즉, g(n)이 n0보다 큰 모든
n에 대해서 항상 f(n)보다 크다는 것)을 보여줌
f(n) = O(g(n))

--- PAGE 85 ---
O(Big-Oh)-표기
 f(n) = 2n2 – 8n + 3
• f(n)의 O-표기는 O(n2)
• f(n)의 단순화된 표현은 n2
• 단순화된 함수 n2 에 임의의 상수
c를 곱한 cn2 이 n이 증가함에
따라 f(n)의 상한이 됨
단, c > 0.
• c=5일 때, f(n) = 2n2-8n+3과 5n2과의 교차점(n0=1/3)이 생기는데, 이 교차점 이후 모든
n에 대해, 즉 n이 무한대로 증가할 때, f(n) = 2n2-8n+3은 5n2보다 절대로 커질 수 없음
• 따라서 O(n2)이 2n2-8n+3의 점근적 상한이 됨

O-표기법 찾는 간단한 방법
“다항식에서 최고 차수 항만을 취한 뒤, 그
항의 계수를 제거하여 g(n)을 정함”

(Big-Omega)-표기
점근적 하한 (Lower Bound)
• 알고리즘의 실행 시간이 "적어도 이만큼은 걸린다"
는 것을 의미
• 알고리즘의 최선의 경우(Best Case) 성능의 한계
를 명시할 때 사용


--- PAGE 88 ---
(Big-Omega)-표기
 Ω-표기의 정의
• 모든 n ≥ n0에 대해서 f(n) ≥ cg(n)이 성립하는 양의 상수 c와 n0가 존재하면, f(n) = Ω(g(n))
 Ω-표기의 의미
• n0 보다 큰 모든 n 대해서 f(n)이 cg(n)보다 작지 않다는 것
 f(n) = Ω(g(n))은 양의 상수를 곱한 g(n)이 f(n)에 미치지 못한다는 뜻
 g(n)을 f(n)의 하한(Lower Bound)이라 함

--- PAGE 89 ---
(Big-Omega)-표기
 f(n) = Ω(g(n))
• n이 증가함에 따라 Ω(g(n))이 점근적 하한이라는 것 (즉, g(n)이 n0보다 큰 모든
n에 대해서 항상 f(n)보다 작다는 것)을 보여줌
f(n) = Ω(g(n))

--- PAGE 90 ---
(Theta)-표기
평균적 성능의 지표
• 상한(O)과 하한(Ω)이 동일할 때 성립하며, 알고리즘의
정확한 증가율을 나타냄
• "이 알고리즘은 정확히 이 정도로 수렴한다"는 가장 강
력한 표기법


--- PAGE 91 ---
(Theta)-표기
 Θ-표기의 정의
• 모든 n ≥ n0에 대해서 c1g(n) ≥ f(n) ≥ c2g(n)이 성립하는 양의 상수 c1, c2, n0가
존재하면, f(n) = Θ(g(n))이다.
 Θ-표기의 의미
• 수행시간의 O-표기와 Ω-표기가 동일한 경우에 사용한다. 즉 동일한 증가율을 의미
 2n2+3n+5=O(n2)과 동시에 2n2+3n+5=Ω(n2) 이므로, 2n2+3n+5=Θ(n2)
 (n2)은 n2과 (2n2+3n+5)이 유사한 증가율을 가지고 있다는 뜻
• 따라서 2n2+3n+5≠Θ(n3) 이고, 2n2+3n+5≠Θ(n)

--- PAGE 92 ---
(Theta)-표기
 f(n)=Θ(g(n))
• n0보다 큰 모든 n에 대해서 Θ-표기가 상한과 하한을 동시에 만족한다는 것을 보여줌
f(n) = Θ(g(n))

(Big-Omega)-표기 예

 n≥1, 3n+3 ≥ 3n  3n+3 = Ω(n)
 n≥1, 100n+6 ≥ 100n  100n+6 = Ω(n)
 n≥1, 10n2+4n+2 ≥ n2  100n2+4n+2 = Ω(n2)
 n≥1, 6*2n+n2 ≥ 2n  6*2n+n2 = Ω(2n)

 3n+3 = Θ(n)
 10n2+4n+2 = Θ(n2)

표기법 간의 상관관계 요약
표기법 의미 (Intuition) 관계식 비유
Big-O (O) 점근적 상한 f(n) ≤ c·g(n) "최대 ~만큼" (≤)
Big-Ω (Ω) 점근적 하한 f(n) ≥ c·g(n) "최소 ~만큼" (≥)
Big-Θ (Θ) 긴밀한 경계 c1·g(n) ≤ f(n) ≤ c2·g(n) "정확히 ~만큼" (=)

자주 사용하는 O-표기
 O(1) 상수 시간(Constant time)
• 입력 크기에 관계없이 항상 일정한 시간 소요.
• 예: 배열 인덱스 접근, 해시 맵 조회.
 O(logn)    로그 시간(Logarithmic time)
• 매 단계에서 검색 범위를 절반씩 줄여나감.
• 예: 이진 탐색 (Binary Search).
 O(n) 선형 시간(Linear time)
• 데이터 양에 비례하여 시간이 증가.
• 예: 단순 루프를 통한 리스트 순회

--- PAGE 98 ---
자주 사용하는 O-표기
 O(nlogn) 로그 선형 시간(Log-linear time)
• 효율적인 정렬 알고리즘의 표준 속도.
• 예: 병합 정렬 (Merge Sort), 퀵 정렬
 O(n2) 이차 시간(Quadratic time)
• 데이터가 증가함에 따라 시간이 제곱으로 증가
• 예: 이중 루프, 버블 정렬
 O(2n) 지수 시간 (Exponential time)
• 입력이 늘어날 때마다 시간이 두 배씩 증가.
• 예: 피보나치 재귀, 외판원 문제(TSP).

자주 사용하는 O-표기
 O(n3) 3차 시간(Cubic time)
 O(nk) 다항식 시간(Polynomial time), k는 상수
