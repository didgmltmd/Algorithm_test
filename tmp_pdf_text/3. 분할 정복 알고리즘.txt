• 총 분할한 횟수 = k라면
• 1번 분할 후 각 입력 크기 n/2

• n/2k = 1일 때 분할 못함
• k = log2n

1. if ( p < q ) { // 배열의 원소의 수가 2개 이상이면

}

시간 복잡도
• 분할하는 부분은 배열의 중간 인덱스 계산과 2회의 순환 호출이므로 O(1) 시간 소요

• 합병의 시간 복잡도 = O(m+n)
• 합병 정렬에서 수행되는 총 비교 횟수
• 각각의 합병에 대해서 몇 번의 비교가 수행되었는지를 계산하여 이들을 모두 합한 수

def merge_sort(arr): 
if len(arr) <= 1: 
return arr
# 1. 분할 (Divide)
mid = len(arr) // 2
left = merge_sort(arr[:mid]) 
right = merge_sort(arr[mid:]) 
# 2. 합병 (Merge) 
return merge(left, right) 
def merge(left, right): 
result = [] 
i = j = 0 
# 두 배열을 비교하며 작은 값을 결과 리스트에 추가 
while i < len(left) and j < len(right): 
if left[i] <= right[j]: 
result.append(left[i])
i += 1 
else: 
result.append(right[j])
j += 1 
# 남은 요소들 추가 
result.extend(left[i:]) 
result.extend(right[j:]) 
return result

def merge_inplace(arr, low, mid, high):
temp = [] 
i, j = low, mid + 1 
while i <= mid and j <= high: 
if arr[i] <= arr[j]: 
temp.append(arr[i]) 
i += 1 
else: 
temp.append(arr[j]) 
j += 1 
while i <= mid: 
temp.append(arr[i])
i += 1

temp.append(arr[j])

for k in range(len(temp)):

def iterative_merge_sort(arr): 
n = len(arr) 
width = 1
while width < n: 
for i in range(0, n, 2 * width): 
# 두 부분 리스트의 범위 계산 
low = i
mid = min(i + width, n) 
high = min(i + 2 * width, n) 
# 병합 수행 (내부 함수 또는 별도 정의 가능) 
merge(arr, low, mid, high)
width *= 2 
return arr
def merge(arr, low, mid, high):
left = arr[low:mid] 
right = arr[mid:high] 
i = j = 0 
k = low 
while i < len(left) and j < len(right): 
if left[i] <= right[j]: 
arr[k] = left[i] 
i += 1 
else: 
arr[k] = right[j] 
j += 1 
k += 1 
# 남은 요소 복사 
while i < len(left): 
arr[k] = left[i] 
i += 1 
k += 1 
while j < len(right): 
arr[k] = right[j] 
j += 1 
k += 1

def partition(arr, low, high): 
pivot = arr[high] # 마지막 원소를 피벗으로 
i = low - 1 # 작은 원소들을 위한 경계 인덱스 
for j in range(low, high): 
if arr[j] <= pivot: 
i += 1
# 교환
arr[i], arr[j] = arr[j], arr[i] 
# 피벗을 중앙으로 이동
arr[i + 1], arr[high] = arr[high], arr[i + 1]
return i + 1

Selection(A, left, right, k)
입력: A[left]~A[right]와 k, 단, 1≤k≤|A|, |A|=right-left+1
출력: A[left]~A[right]에서 k 번째 작은 원소

피봇과 A[left]의 자리를 바꾼 후,
피봇과 배열의 각 원소를 비교하여 피봇보다 작은 숫자는 A[left]~A[p-1]로 옮기고,

4. else if ( k = S + 1 ) return A[p] // 피봇 = k번째 작은 숫자
5. else Selection(A, p+1, right, k-S-1)    // Large group에서 찾기

import random 
def median_of_medians(arr, k): 
# 원소가 적으면 정렬해서 반환 
if len(arr) <= 5: 
return sorted(arr)[k-1] 
# 1. 5개씩 그룹화하여 각 그룹의 중앙값 찾기 
sublists = [arr[i:i+5] for i in range(0, len(arr), 5)] 
medians = [sorted(sub)[len(sub)//2] for sub in sublists] 
# 2. 중앙값들의 중앙값(Pivot)을 재귀적으로 찾기 
pivot = median_of_medians(medians, len(medians)//2 + 1) 
# 3. 피벗 기반 분할 
lows = [x for x in arr if x < pivot] 
highs = [x for x in arr if x > pivot] 
pivots = [x for x in arr if x == pivot]
if k <= len(lows): 
return median_of_medians(lows, k) 
elif k <= len(lows) + len(pivots): 
return pivots[0] 
else: 
return median_of_medians(highs, k - len(lows) - len(pivots))

요약
• 분할 정복 (Divide-and-Conquer) 알고리즘: 주어진 문제의 입력을 
분할하여 문제를 해결 (정복)하는 방식의 알고리즘
• 합병 정렬 (Merge sort): n개의 숫자들을 n/2개씩 2개의 부분 문제
로 분할하고, 각각의 부분 문제를 재귀적으로 합병 정렬한 후, 2개의 정
렬된 부분을 합병하여 정렬 (정복)함.   시간 복잡도는 O(n log n)

--- PAGE 138 ---
요약
• 합병 정렬의 공간 복잡도는 O(n)
• 퀵 정렬 (Quick sort): 피봇 (pivot)이라 일컫는 배열의 원소를 기준
으로 피봇보다 작은 숫자들은 왼편으로, 피봇보다 큰 숫자들은 오른편에 
위치하도록 분할하고, 피봇을 그 사이에 놓음
• 퀵 정렬은 분할된 부분 문제들에 대하여서도 위와 동일한 과정을 순환으
로 수행하여 정렬

--- PAGE 139 ---
요약
• 퀵 정렬의 평균 경우 시간 복잡도는 O(n log n), 최악 경우 시간 복잡도
는 O(n2), 최선 경우 시간 복잡도는 O(n log n)
• 선택 (Selection) 문제: k 번째 작은 수를 찾는 문제로서, 입력에서 퀵 
정렬에서와 같이 피봇을 선택하여 피봇보다 작은 부분과 큰 부분으로 분
할한 후에 k 번째 작은 수가 들어있는 부분을 순환으로 탐색함
• 평균 경우 시간 복잡도는 O(n)

• 최근접 점의 쌍 (Closest Pair) 문제: n개의 점들을 1/2로 분할하여

• 시간 복잡도는 O(nlog2n)
